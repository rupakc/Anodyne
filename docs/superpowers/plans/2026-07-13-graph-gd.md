# GD — Graph-aware evaluation (MoE judges)

**Branch:** `feat/graph-gd-eval` (off `main`). Wave 1 task on top of GA's frozen
interfaces (`docs/superpowers/specs/2026-07-13-graph-modality-design.md` §6, §9-GD).

## Goal

Six graph expert judges behind the existing `Judge` port, dispatched by the
evaluator when a dataset version's modality is `Modality.GRAPH`, aggregated by the
existing `WeightedAggregator` into the same 360-degree JSON+HTML report. Tabular/
text judges are untouched for those modalities.

## Judges (package `anodyne_evaluation.graph_judges`)

| Class | Dimension | Default weight | Needs reference? | LLM? |
|---|---|---|---|---|
| `StructuralFidelityGraphJudge` | `GRAPH_STRUCTURE` | 0.25 | yes | no |
| `OntologyConsistencyGraphJudge` | `GRAPH_ONTOLOGY` | 0.20 | no | no |
| `GraphPrivacyJudge` | `GRAPH_PRIVACY` | 0.15 | yes | no |
| `ConnectivityCoverageGraphJudge` | `GRAPH_CONNECTIVITY` | 0.15 | no | no |
| `GraphUtilityJudge` | `GRAPH_UTILITY` | 0.15 | yes | no |
| `SemanticPlausibilityGraphJudge` | `GRAPH_SEMANTIC` | 0.10 | no | yes (`LLMProvider`) |

`GRAPH_WEIGHTS` sums to 1.0; the aggregator renormalizes over whichever
dimensions applied (judges that need a missing reference raise `JudgeNotApplicable`).

### Metrics summary
- **Structural**: degree-distribution KS, clustering/assortativity/modularity/
  avg-path-length deltas, and the scalable **normalized-Laplacian eigenvalue
  (spectral) distance** — NOT graph edit distance / isomorphism (NP-hard).
- **Ontology**: pure-Python domain/range + datatype/required/choice validation
  pass-fraction (justified over pySHACL: the LPG isn't RDF; SHACL would need a
  lossy RDF projection to re-check rules the ontology already states natively;
  relation cardinality isn't in the frozen GA ontology model → documented no-op).
- **Connectivity**: giant-component fraction, isolated-node fraction, relation-
  and node-type coverage vs ontology (intrinsic — no reference).
- **Utility (GNN TSTR)**: seeded sklearn `RandomForestClassifier` predicting node
  type from structural features (degree, clustering, triangles, avg-neighbour
  degree, community size) + shared numeric node properties; efficacy = TSTR/TRTR.
- **Privacy**: structural re-identification rate (node fingerprint match) +
  verbatim-edge overlap vs reference → `score = 1 - leakage` (heuristic; DP is a
  future track). An identical copy scores near 0; a resampled graph scores high.
- **Semantic**: LLM rates sampled triples for realism + coherence (1-5), mocked
  in tests.

## Dispatch

`evaluator.judges_for_modality(modality, provider, model_config)` returns
`graph_judges(...)` for `Modality.GRAPH`, else `default_judges(...)`. The
evaluation activity loads a `graph_json` artifact via `loader.load_graph` →
`GraphDataset` (bypassing the columnar `load_artifact` path), places it on the
new `EvaluationContext.subject_graph`/`reference_graph` fields (leaving `subject`
an empty DataFrame), and selects judges by modality.

## Deps
`networkx>=3.2` (BSD) + `anodyne-graph` (workspace) added to
`anodyne-evaluation/pyproject.toml`. pySHACL NOT added (see above). scipy/sklearn
already present.

## Tests
`test_graph_judge_{structural,ontology,semantic,connectivity,utility,privacy,
dispatch}.py` + `conftest.py` (fixtures). Known-range assertions (identical graphs
→ fidelity ~1 / privacy ~0; injected edges → lower fidelity; ontology violations
→ lower pass-rate), aggregation/weight renormalization, LLM mocked, seeded.
