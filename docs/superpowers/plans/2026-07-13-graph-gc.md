### Task 1: `GraphExporter` module — RDF/semantic formats (Turtle, N-Triples, JSON-LD, RDF/XML) + OWL

**Files:** `packages/anodyne-graph/src/anodyne_graph/export.py`,
`packages/anodyne-graph/tests/test_graph_export_rdf.py`,
`packages/anodyne-graph/tests/test_graph_export_owl.py`.

- [x] `dataset_to_rdf(GraphDataset) -> rdflib.Graph`: nodes -> `rdf:type onto:<NodeType>` +
  datatype-property triples; edges -> a direct `(subject onto:<EdgeType> object)` triple.
- [x] Edge properties: **standard RDF reification** (`rdf:Statement`/`subject`/`predicate`/
  `object`), not RDF-star — round-trips through every serializer here without a star-aware parser.
  Reification bnode id is deterministic (`stmt-<edge.id>`), not `BNode()`, so output/tests are
  stable across runs.
- [x] `ontology_to_owl(GraphOntology) -> rdflib.Graph`: node types -> `owl:Class` (+
  `rdfs:subClassOf` from `subclass_of`); edge types -> `owl:ObjectProperty` with
  `rdfs:domain`/`range`; each `PropertySpec` -> `owl:DatatypeProperty` (`onto:<Type>.<prop>`) with
  `rdfs:range` from a `datatype -> xsd:*` table (default `xsd:string`).
- [x] `encode_dataset(dataset, fmt)` dispatches `ttl`/`nt`/`jsonld`/`rdfxml`/`owl`.
- [x] Tests: parse every RDF serialization back with `rdflib.Graph().parse(...)`, assert triple
  count matches and key triples are present (node types, edge triple, reification statement,
  OWL classes/properties/domain/range). Determinism asserted directly (byte-equal across calls).

### Task 2: Property-graph formats (GraphML, GEXF, Cypher, Neo4j admin-import CSVs, node-link JSON)

**Files:** same `export.py`; `packages/anodyne-graph/tests/test_graph_export_property_graph.py`.

- [x] `dataset_to_networkx -> nx.MultiDiGraph`: all node/edge properties **stringified** (JSON for
  list/dict) before handing to networkx, since GraphML/GEXF require a homogeneous attribute type
  per key across all nodes/edges — a heterogeneous ontology would otherwise crash the writer.
  `directed` (declared on `EdgeType`, not the `Edge` instance) is looked up via
  `ontology.edge_type(edge.type)`, default `True` if the type isn't declared.
- [x] `graphml`/`gexf` via `nx.write_graphml`/`nx.write_gexf` into an in-memory `BytesIO`.
- [x] `cypher`: one `CREATE (:<Type> {id, ...props})` per node; one
  `MATCH (a {id: ...}), (b {id: ...}) CREATE (a)-[:<Type> {...}]->(b)` per edge — matched by `id`
  property, no label assumed (so a dangling/mismatched type never breaks the match). Nodes/edges
  sorted by `id` for determinism.
- [x] `neo4j-csv`: one `nodes_<Type>.csv` (`id:ID,:LABEL,<props>`) + one `edges_<Type>.csv`
  (`:START_ID,:END_ID,:TYPE,<props>`) **per type** (properties differ per type; a single universal
  header would drop or blank columns), zipped in-memory (`zipfile`).
- [x] `graph_json`: passthrough via the existing `to_json_bytes`/`from_json_bytes`.
- [x] Tests: `nx.read_graphml`/`read_gexf` round-trip + node/edge counts and a `type` attribute;
  Cypher output checked for exact `CREATE`/`MATCH...CREATE` line counts and content; Neo4j zip
  checked for expected file names + exact CSV headers/row counts; `graph_json` round-trip equality.

### Task 3: GNN formats (edge-index + node-feature `.npz`, edge-list `graph-parquet`)

**Files:** same `export.py`; `packages/anodyne-graph/tests/test_graph_export_gnn.py`.

- [x] `.npz`: `edge_index` (2, E) int64 in node-index space (order = `dataset.nodes`), a one-hot
  `node_features` (N, num_node_types) matrix (no numeric-property tensorization in GC — a
  documented follow-up), `node_ids`/`node_type_names`/`edge_type_names` vocabularies to decode ids.
  Edges whose endpoints aren't in `nodes` are dropped (best-effort), never raise.
- [x] `graph-parquet`: edge list (`edge_id, source, target, type, directed`) as a single
  `pyarrow.Table` written with `pq.write_table` — one shot, not per-row Python objects.
  Named `graph-parquet` (not `parquet`) to avoid colliding with the tabular exporter's format key.
- [x] Tests: `np.load` round-trip shape/content assertions + determinism; dangling-edge-drop case;
  `pq.read_table` round-trip column/row assertions.

### Task 4: `GraphExporter` (the `Exporter` port adapter) + export-route dispatch

**Files:** `export.py` (`GraphExporter` class); `apps/api-gateway/src/api_gateway/export_routes.py`,
`apps/api-gateway/src/api_gateway/downloads.py`, `apps/api-gateway/src/api_gateway/deps.py`;
`packages/anodyne-graph/tests/test_graph_export_exporter.py`,
`apps/api-gateway/tests/test_export_routes_graph.py`.

- [x] `GraphExporter(Exporter)`: same shape as `PyArrowExporter` — `store.get(artifact_uri)` ->
  `from_json_bytes` -> `encode_dataset` -> `store.put(key, encoded)` -> `ExportArtifact` (
  `row_count` = `len(nodes) + len(edges)`). Default format (no `format` given) is `graph_json`
  passthrough. Unsupported format -> `UnsupportedGraphExportFormatError` (`anodyne_graph.errors`).
- [x] `deps.get_graph_exporter()` added (real `GraphExporter()`), alongside the existing
  `deps.get_exporter()` (`PyArrowExporter()`) — both overridable in tests.
- [x] `export_routes.py`: `SUPPORTED_FORMATS` is now the **union** of
  `anodyne_export.exporter.SUPPORTED_FORMATS` and `anodyne_graph.export.GRAPH_SUPPORTED_FORMATS`
  (request-level 400 validation doesn't yet know the version's modality). Dispatch:
  `active_exporter = graph_exporter if version.format == "graph_json" else exporter` (a graph
  version's stored artifact format is always `graph_json` — see `GraphHandler.artifact_format`).
  Streaming/`Content-Disposition` behavior unchanged; no presigned URLs.
- [x] `downloads.py`: added media-type/extension entries — `ttl`->`text/turtle`/`.ttl`,
  `nt`->`application/n-triples`/`.nt`, `jsonld`->`application/ld+json`/`.jsonld`,
  `rdfxml`->`application/rdf+xml`/`.rdf`, `owl`->`application/rdf+xml`/`.owl`,
  `graphml`->`application/xml`/`.graphml`, `gexf`->`application/xml`/`.gexf`,
  `cypher`->`text/plain`/`.cypher`, `neo4j-csv`->`application/zip`/`.zip`,
  `npz`->`application/octet-stream`/`.npz`, `graph-parquet`->`application/octet-stream`/`.parquet`.
- [x] Tests: `GraphExporter.export` parametrized over every `GRAPH_SUPPORTED_FORMATS` value against
  a fake `ObjectStore`; unsupported-format raise; row-count check. Gateway: graph-version request
  dispatches to `graph_exporter` (tabular exporter untouched) and vice versa; a graph-only format
  string passes the merged 400 gate; one end-to-end test swaps in the real `GraphExporter` and
  asserts a real `graphml` document streams back with the right content-type/filename.

## Follow-ups (not in GC scope, flagged for later waves)

- GNN `.npz` node features are a pure one-hot of node type; tensorizing numeric `PropertySpec`
  values into `node_features` is a natural GB/GD follow-up once generation emits richer numeric
  properties at scale.
- RDF-star was considered for edge properties and rejected in favor of standard reification (see
  Task 1) for broader parser compatibility across the four RDF serializations GC supports.

## Note: cross-modality format mismatch

`SUPPORTED_FORMATS` in `export_routes.py` is the *union* of tabular and graph formats (checked
before the version's modality is known). A request like `format=csv` against a graph version (or
`format=ttl` against a tabular one) passes that gate but is invalid for the exporter it dispatches
to; both `PyArrowExporter`/`GraphExporter` raise their own `Unsupported*FormatError` for it, which
`export_version` now catches and turns into a `400` (not an unhandled `500`) — see
`export_routes.py`'s `try/except` around the `active_exporter.export(...)` call.
