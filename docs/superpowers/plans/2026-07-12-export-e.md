# Export & Storage (E) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Checkbox (`- [ ]`) tracking.

**Goal:** `Exporter` port + `anodyne-export` adapter (CSV/JSON/Parquet/Arrow, chunked via pyarrow,
size-based format default), `export_artifacts` table + repo, gateway route
`POST /datasets/{id}/versions/{id}/export` in a focused module.

**Architecture:** see `docs/superpowers/specs/2026-07-12-export-e-design.md`.

**Tech stack:** unchanged (Python 3.12, pydantic v2, pyarrow>=17, SQLAlchemy async, Alembic,
FastAPI, moto for offline S3 tests).

  sources, mirroring `anodyne-tabular`'s `pyproject.toml` shape.

  - `resolve_format(10, None) == "csv"`
  - `resolve_format(500_000, None) == "csv"` (boundary: not `>`)
  - `resolve_format(500_001, None) == "parquet"`
  - `resolve_format(2_000_000, "json") == "json"` (explicit always wins)
  - `LARGE_DATASET_ROW_THRESHOLD == 500_000` (named-constant assertion).
- [ ] Write failing round-trip tests (one file per format, small ~50-row fixture table with an
  int column, a float column, a nullable string column with some `None`s, and a unicode string
  value): build fixture `pa.table(...)`; write it to an in-memory fake `ObjectStore` (a tiny
  dict-backed fake implementing `anodyne_core.ports.ObjectStore`, local to
  `packages/anodyne-export/tests/`, not `moto` — no S3 semantics needed for this layer) as Parquet
  bytes (`pq.write_table`); construct a `DatasetVersion(artifact_uri=..., format="parquet",
  row_count=50, ...)`; call `await PyArrowExporter().export(version, fake_store, format="csv"|...)`;
  fetch the uploaded bytes from the fake store; parse back with the matching reader; assert every
  row/column matches the fixture (nulls preserved, unicode preserved, floats within tolerance).
- [ ] Write failing `test_export_unsupported_format.py`: `format="xml"` → raises
  `UnsupportedExportFormatError` before any I/O.
- [ ] Write failing `test_export_batching.py`: 25-row fixture, `batch_size=10` →
  `export(..., batch_size=10)`; assert the writer processed ≥ 3 batches (spy via monkeypatching
  `pyarrow.parquet.ParquetFile.iter_batches` to record call `batch_size`, or count batches yielded
  by wrapping the real iterator) — proves chunking, not a single-shot `pq.read_table()`.
- [ ] Run all → fail (`ModuleNotFoundError`).
- [ ] Implement `exporter.py`:
  - `LARGE_DATASET_ROW_THRESHOLD = 500_000`
  - `SUPPORTED_FORMATS = frozenset({"csv", "json", "parquet", "arrow"})`
  - `UnsupportedExportFormatError(ValueError)`
  - `resolve_format(row_count: int, requested: str | None) -> str`
  - `PyArrowExporter(Exporter)`:
    - `export(...)`: validate format via `resolve_format` + `SUPPORTED_FORMATS` membership (raise
      `UnsupportedExportFormatError` early); `data = await store.get(version.artifact_uri)`;
      `pf = pq.ParquetFile(io.BytesIO(data))`; dispatch to `_write_csv`/`_write_json`/
      `_write_parquet`/`_write_arrow(pf, batch_size)` returning `bytes`; build the export key
      `datasets/{dataset_id}/{version_id}/export.{ext}`; `await store.put(key, bytes)`; return an
      `ExportArtifact` (caller/route fills `id`/`tenant_id` via a thin wrapper if needed — decide
      exact field-population split while implementing Task 1's `Exporter` signature; keep
      `PyArrowExporter` free of `DatasetRepository`/tenant-id concerns per the spec).
    - `_write_parquet`: `pq.ParquetWriter(buf, pf.schema_arrow)`; loop
      `pf.iter_batches(batch_size=batch_size)`, `writer.write_batch(b)`; `writer.close()`.
    - `_write_arrow`: `pa.ipc.new_stream(buf, pf.schema_arrow)`; same loop; `writer.close()`.
    - `_write_csv`: `pyarrow.csv.CSVWriter(buf, pf.schema_arrow)`; same loop; `writer.close()`.
    - `_write_json`: loop batches, `for row in batch.to_pylist(): buf.write(json.dumps(row).encode() + b"\n")`.
- [ ] Run tests → pass. `ruff`/`mypy` clean.
- [ ] Commit: `feat(export): add PyArrowExporter with chunked CSV/JSON/Parquet/Arrow writers`.

## Self-review

**Spec coverage:** `Exporter` port + `ExportArtifact` → T1 ✓; chunked CSV/JSON/Parquet/Arrow +
size-based default (named constant, boundary tested) → T2 ✓; workspace registration → T3 ✓;
`export_artifacts` table/migration/RLS/repo → T4 ✓; gateway route in a focused module, tenant
ownership check, `datasets:read` gating → T5 ✓; full-suite + self-review → T6 ✓.

**Compute-placement justification:** captured explicitly in the design spec's Decisions table
(no Ray/Temporal — chunked in-process pyarrow is sufficient for the I/O-bound export workload;
`run_export`/`PyArrowExporter.export` is structured as a plain function so a Ray-remote wrapper is
a drop-in follow-up, not a redesign).

**Placeholders:** none load-bearing; Task 4's port-location note (whether `ExportRepository` lands
in Task 1 or Task 4) is a sequencing choice, not a missing decision — either location satisfies the
spec's "sibling port" requirement.
