# GA — Graph modality core (walking skeleton) — Plan

Branch: `feat/graph-ga-core`. Implements only the §9 "GA" bullet of
`docs/superpowers/specs/2026-07-13-graph-modality-design.md`: one end-to-end
happy path — description → ontology (LLM) → LLM-generated property graph →
node-link JSON artifact → download. LLM is always behind `LLMProvider`; no new
heavy deps (no networkx/rdflib in GA).

## Deliverables
1. `Modality.GRAPH` added to `anodyne_dataset.models.Modality`.
2. New package `packages/anodyne-graph` (src layout, mirrors `anodyne-text`):
   - `models.py` — Pydantic domain models (frozen public shapes below).
   - `serialization.py` — node-link JSON `to_json_bytes` / `from_json_bytes`.
   - `ports.py` — `OntologyProposer` Protocol (no adapter imports).
   - `ontology.py` — `LLMOntologyProposer` (description → `GraphOntology`).
   - `generator.py` — `LLMGraphGenerator` (ontology + shard + seed → `GraphDataset`).
3. `anodyne-workflows`: `GraphHandler` in `handlers.py`, registered as `"graph"`
   in `modality.py`. Depends on `anodyne-graph` + `anodyne-llm`.
4. Gateway: `deps.get_ontology_proposer`; `POST /datasets` graph branch;
   generate route default-model pick extended to graph.

## Frozen public interfaces (Wave 1 builds on these)

### Domain models (`anodyne_graph.models`)
- `PropertySpec(name: str, datatype: str, nullable: bool = False, constraints: dict[str, Any] = {})`
  — datatype is a plain string: "string"/"integer"/"float"/"boolean"/"datetime".
- `NodeType(name: str, properties: list[PropertySpec] = [])`
- `EdgeType(name: str, source_type: str, target_type: str, properties: list[PropertySpec] = [], directed: bool = True)`
- `GraphOntology(node_types: list[NodeType] = [], edge_types: list[EdgeType] = [], subclass_of: dict[str, str] = {})`
- `Node(id: str, type: str, properties: dict[str, Any] = {})`
- `Edge(id: str, type: str, source: str, target: str, properties: dict[str, Any] = {})`
- `GraphDataset(ontology: GraphOntology, nodes: list[Node] = [], edges: list[Edge] = [], metrics: dict[str, Any] = {})`

### node-link artifact JSON (`artifact_format = "graph_json"`)
```json
{
  "ontology": { "node_types": [...], "edge_types": [...], "subclass_of": {} },
  "nodes":   [ {"id": "...", "type": "...", "properties": {...}} ],
  "edges":   [ {"id": "...", "type": "...", "source": "...", "target": "...", "properties": {...}} ],
  "metrics": { "node_count": N, "edge_count": M,
               "nodes_by_type": {..}, "edges_by_type": {..} }
}
```
`to_json_bytes(GraphDataset) -> bytes`, `from_json_bytes(bytes) -> GraphDataset` round-trip exactly.

### Ports (`anodyne_graph.ports`)
```python
class OntologyProposer(Protocol):
    async def propose(self, spec: DatasetSpec, provider: LLMProvider,
                      config: ModelConfig) -> GraphOntology: ...
```
Stateless, per the design spec's literal signature. Graph *generation* uses
`LLMGraphGenerator.generate(spec, start_index, count, seed, shard_index) -> GraphDataset`
(same `(spec, start, count, seed)` shape as the `Generator` port; returns a
`GraphDataset` rather than a pyarrow table, since a graph is not columnar).

### Handler
`GraphHandler` — `shard_rows` large (one shard for GA), `artifact_format = "graph_json"`,
`generate_shards` (invokes `LLMGraphGenerator` with the tenant model + a
provider built from `ctx.secret_store`), `assemble` (merges shard JSON into one
`GraphDataset`, uploads one `graph_json` artifact). Registered as `"graph"`.

### Gateway
- `deps.get_ontology_proposer` -> `OntologyProposalHandle(proposer, provider, config)`
  (mirrors `get_schema_proposer`; picks the Gemini default via `pick_default_model`).
- `POST /datasets` graph branch: build spec, propose ontology, store JSON in
  `spec.directives["ontology"]`, return spec (includes the ontology).
- Generate route default-model pick now covers `Modality.GRAPH` too.

## Key decisions
- Ontology stored in `DatasetSpec.directives["ontology"]` — no DB table/migration.
- `OntologyProposer.propose` takes the full `DatasetSpec` (richer than
  `SchemaProposer`: description + directives) and provider+config per-call
  (stateless port), exactly as the design spec froze it.
- `get_ontology_proposer` returns a small handle bundling proposer+provider+config
  so the route needs no extra LLM deps (avoids blast radius: `get_llm_provider`
  can't be an unconditional dep — it fails when `secret_key` is unset in tests).
  The handle is overridden in the two gateway test fixtures that POST /datasets.
- Generator is deterministic: `np.random.default_rng([seed, shard_index])`,
  Faker seeded, LLM temperature=0. PII-looking node properties always faked.
  Referential integrity enforced (edges only join existing nodes whose types
  match the `EdgeType` domain/range); nodes deduped by (type, natural key).
- `LLMGraphGenerator` does not subclass the pyarrow `Generator` (graph is JSON,
  not columnar) — GB engines implement the same `generate(...)->GraphDataset` shape.
