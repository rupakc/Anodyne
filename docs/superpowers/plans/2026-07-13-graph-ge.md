# GE ‚Äî Graph modality Web UI

Branch `feat/graph-ge-ui`, off `main`. Work confined to `apps/web`. No servers run;
offline checks only (typecheck / lint / test / build).

## Scope (spec ¬ß7, roadmap ¬ß9 "GE")
1. Graph wizard added to the modality chooser (`graph-wizard.tsx`): describe domain ‚Üí
   review/edit proposed ontology ‚Üí node count + topology ‚Üí generate. Sources:
   from-description, upload sample (GraphML/JSON), provide ontology. Degrades gracefully
   when a backend route is absent (404/501 tolerated).
2. `components/graph-explorer.tsx` ‚Äî self-contained Canvas 2D force-directed viz. NO
   external dep. Samples large graphs (top-N by degree) + full-graph aggregate stats from
   `metrics`. Nodes colored by type, edges labeled by type, click shows properties.
3. `components/ontology-view.tsx` ‚Äî node types + properties, edge types source‚Üítarget,
   subclass hierarchy.
4. Export picker extended for graph formats (Turtle, JSON-LD, GraphML, Cypher, Neo4j CSV,
   OWL, node-link JSON) via the streaming `downloadToFile` helper.
5. Evaluation report already renders generic `ExpertScore`s ‚Üí graph dimensions display fine.

## Backend contract consumed
- `POST /datasets` `{modality:"graph", description, directives}` ‚Üí `directives.ontology`.
- `PATCH /datasets/{id}` `{directives}` ‚Äî ontology edit before generate.
- `POST /datasets/{id}/generate` (`model_config_id`, `require_review`) ‚Üí poll `/jobs/{id}`.
- `GET .../versions/{vid}/download` ‚Äî node-link JSON artifact (fetched + parsed for the explorer).
- `POST .../versions/{vid}/export?format=` ‚Äî graph formats via streaming download.
- Evaluate ‚Äî existing flow; graph judges surface as generic experts.

## Self-containment + large graphs
The explorer is a pure-TS inline Verlet/force loop drawn to `<canvas>` ‚Äî zero imports, CSP-safe.
Layout runs a fixed iteration budget in a `requestAnimationFrame` loop, then idles. Graphs above
a node cap are reduced to the top-N highest-degree nodes plus induced edges; the header always
shows true full-graph node/edge counts and per-type breakdowns from `metrics`. `getContext`
returning null (jsdom / no-canvas) is guarded, and an accessible node/edge summary list is always
rendered so the component is testable and screen-reader friendly.

## Endpoints stubbed / tolerated

- "Upload sample" / "provide ontology" sources wire the fields; from-sample generation may land
