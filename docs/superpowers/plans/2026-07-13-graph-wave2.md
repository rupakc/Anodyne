## Follow-up fixes (done first, on `main`)

### FIX-1: Spectral distance inflated for unequal-size graphs
`packages/anodyne-evaluation/.../graph_judges/structural.py` `_spectral_distance`: replace zero-padding of the shorter spectrum with **resampling both spectra to a common fixed length** (linear interpolation over the sorted eigenvalues), so a node-count gap no longer dominates the structural signal. Keep the [0,1] normalization. Failing test: two graphs of very different sizes but similar structure (e.g. BA(50) vs BA(200)) should score a *smaller* distance than under zero-padding, and identical graphs still score 0.

### FIX-2: Empty-graph ontology score = 1.0
`packages/anodyne-evaluation/.../graph_judges/ontology.py`: when `checks == 0` (no nodes/edges), raise `JudgeNotApplicable("ontology consistency requires a non-empty graph")` instead of returning `1.0`. Failing test: empty `GraphDataset` → judge raises `JudgeNotApplicable`.

### FIX-3: Cypher control-character escaping
`packages/anodyne-graph/.../export.py` `_cypher_value`: after escaping `\` and `"`, also escape control characters — `\n`→`\\n`, `\r`→`\\r`, `\t`→`\\t`, and any remaining C0 control char → `\\uXXXX`. Failing test: a node property value containing `"\n\t"` produces a Cypher literal with no raw control chars.

### FIX-4: RDF datetime literal typed as xsd:dateTime
`packages/anodyne-graph/.../export.py` `dataset_to_rdf`: type A-Box property literals using the property's **declared datatype** from the ontology (look up the node/edge type's `PropertySpec.datatype`; `datetime` → `Literal(value, datatype=XSD.dateTime)`), so instance data agrees with the OWL T-Box `rdfs:range xsd:dateTime`. Pass the declared datatype into `_literal`. Failing test: a node with a datetime-typed property serializes an A-Box literal whose datatype is `xsd:dateTime`.

## GF — Ontology mapping / alignment (worktree `feat/graph-gf-mapping`)

**Goal:** Given two `GraphOntology` objects (source + target), propose entity/relation alignments via a hybrid matcher (lexical + embedding + LLM), emit **SSSOM** mappings, and route low-confidence mappings through HITL review.

**Files (create under `packages/anodyne-graph/src/anodyne_graph/mapping/`):**
- `models.py` — `MappingRelation` (StrEnum: `exact_match`/`close_match`/`broad_match`/`narrow_match`/`related_match`, matching SKOS/SSSOM predicates), `Mapping` (subject_id, predicate, object_id, confidence: float, justification: str, matcher: str), `MappingSet` (source/target ontology ids, list[Mapping], metadata).
- `matchers.py` — `LexicalMatcher` (normalized string / token-Jaccard / edit-distance similarity, deterministic, no deps beyond stdlib+numpy), `EmbeddingMatcher` (uses `LLMProvider.embed` if available via the port; cosine similarity; deterministic given fixed embeddings), `LLMMatcher` (asks the tenant LLM to adjudicate candidate pairs, returns predicate+confidence+justification). Each matcher scores candidate `(source_entity, target_entity)` pairs.
- `aligner.py` — `OntologyAligner`: orchestrates matchers (lexical prefilter → embedding rerank → LLM adjudication on the top-k borderline pairs), applies a confidence threshold, produces a `MappingSet`. Alignments at/above `auto_accept` threshold are accepted; those in `[review_floor, auto_accept)` are flagged `needs_review`; below `review_floor` are dropped. Aligns both node types and edge types.
- `sssom.py` — `to_sssom_tsv(mapping_set) -> bytes` (SSSOM TSV: `subject_id`, `predicate_id`, `object_id`, `mapping_justification`, `confidence`, `subject_label`, `object_label`) and `to_sssom_json(mapping_set) -> bytes`. Predicate IRIs use `skos:` CURIEs.
- `ports.py` — `EntityMatcher` protocol (`score(source, target) -> float`), so the aligner depends on the protocol not concrete matchers.

**HITL integration:** low-confidence mappings become review items. Reuse `anodyne-hitl` `ReviewTask`/annotation model — do NOT add a new table if the existing `review_tasks` schema fits (a review task referencing the mapping-set artifact + the specific mapping). If a new field is genuinely needed, add it via a linear Alembic migration from head with `tenant_id` + RLS. Prefer: serialize the `MappingSet` as an artifact, create one `ReviewTask` per `needs_review` mapping (or one per mapping-set with the flagged list in its payload — pick the lighter option and document it).

**Determinism:** matchers seeded; sorted candidate iteration; stable tie-breaking by `(subject_id, object_id)`. LLM/embedding calls only through the `LLMProvider` port; tests inject a fake provider.

**Tests:** lexical matcher similarity values; aligner threshold routing (auto-accept / needs-review / drop); SSSOM TSV+JSON round-trip and schema columns; deterministic output across two runs with the same seed; HITL review-task creation for flagged mappings.

## GG — GraphRAG multi-hop QA fixtures (worktree `feat/graph-gg-graphrag`)

**Goal:** From a generated `GraphDataset`, synthesize a **GraphRAG evaluation fixture**: multi-hop questions, each grounded in a concrete path through the graph, with a gold answer and the gold supporting subgraph/path.

**Files (create under `packages/anodyne-graph/src/anodyne_graph/graphrag/`):**
- `models.py` — `QAPath` (ordered list of `(node_id, edge_id)` hops + terminal node), `GraphQAItem` (question: str, answer: str, answer_node_ids: list[str], gold_path: QAPath, hop_count: int, question_type: str, difficulty: str), `GraphQAFixture` (dataset_version_id, list[GraphQAItem], metadata incl. seed).
- `pathfinder.py` — deterministic sampler of connected multi-hop paths of length `k` (2–4 hops) from the graph using seeded RNG over sorted adjacency; avoids trivial cycles; returns `QAPath`s spanning declared relation types.
- `question_gen.py` — `GraphRAGGenerator`: for each sampled path, build a templated natural-language multi-hop question (template families: chained-relation lookup, aggregation/count over a neighborhood, existence/negation, comparison) AND optionally refine phrasing via the tenant LLM (through `LLMProvider`; fake in tests). The **answer is derived from the graph, never from the LLM** — the LLM only rewrites the question surface form. Gold answer + supporting path come from the graph traversal, guaranteeing groundedness.
- `export.py` — `fixture_to_jsonl(fixture) -> bytes` (one JSON object per QA item, GraphRAG-eval friendly), and a `graphrag_manifest` describing the fixture.

**Determinism & grounding:** path sampling and template selection seeded; every answer is computed from the graph so it is verifiably correct; no hallucinated facts (LLM confined to surface rewriting, and if the fake/real LLM output is empty the template phrasing is used).

**Tests:** pathfinder returns valid connected paths of requested hop count (each hop is a real edge); answers match graph ground truth; question templates cover all families; deterministic fixture across two seeded runs; export JSONL schema; empty/too-small graph raises a clear error.

## GH — Graph perturbations (worktree `feat/graph-gh-perturb`)

**Goal:** Extend sub-system D with graph perturbations: structural (edge rewire, edge dropout, node dropout, degree-preserving shuffle) and semantic (ontology-violation injection — wrong edge endpoints/types, missing required properties, out-of-range property values).

**Key architectural note:** the perturbation activity (`packages/anodyne-workflows/.../perturbation_activities.py`) currently round-trips artifacts through a `pyarrow.Table` (`_read_table`/`_write_table`, parquet/jsonl only). Graph artifacts are node-link `graph_json` (see `anodyne_graph.serialization`). Add a **graph-aware branch** in the activity: when the parent version's `format` is `graph_json` (or the dataset modality is GRAPH), load via `anodyne_graph.serialization.from_json_bytes` → `GraphDataset`, apply a graph perturbator, serialize back via `to_json_bytes`. Mirror how `evaluation_activities` added a `Modality.GRAPH` branch with `load_graph`. Do NOT force graphs through `pa.Table`.

**Files:**
- Extend `packages/anodyne-dataset/.../models.py` `PerturbationFamily` — add graph families: `GRAPH_REWIRE`, `GRAPH_DROPOUT`, `GRAPH_ONTOLOGY_VIOLATION` (values kebab/snake consistent with existing `noise`/`drift`).
- Create `packages/anodyne-graph/src/anodyne_graph/perturb.py` — `perturb_graph(dataset: GraphDataset, family, intensity: float, seed: int, params: dict) -> GraphDataset`: seeded, deterministic; `intensity` in [0,1] scales the fraction of edges/nodes affected; rewire preserves node set and (optionally) degree; dropout removes a fraction of edges/nodes (and dangling edges); ontology-violation injects a controlled number of violations the ontology judge will detect. Never mutate the input dataset (return a new one). PII in any newly-synthesized property stays faked.
- Wire dispatch: register the graph perturbator so `RegistryPerturbator` routes `Modality.GRAPH` to `perturb_graph` (follow the existing `anodyne_perturbation.handlers`/`registry` pattern; the graph handler lives in `anodyne-perturbation` and imports `anodyne_graph.perturb`, OR is injected — keep `anodyne-core` clean). Add the graph branch in `perturbation_activities.py`.

**Determinism:** seeded RNG over sorted nodes/edges; stable selection of affected elements; identical output across two runs with the same seed.

**Tests:** each family at intensity 0 is a no-op (graph unchanged); intensity 1 affects the expected fraction; rewire preserves node count (and degree when degree-preserving); dropout removes the expected fraction and leaves no dangling edges; ontology-violation injection makes the `OntologyConsistencyGraphJudge` score drop measurably; deterministic across seeds; the perturbation activity round-trips a `graph_json` artifact end-to-end (unit-level with fakes, offline).
