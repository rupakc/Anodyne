# GB — Graph generation engines — Plan

Branch: `feat/graph-gb-engines` (off `main`). Implements the §9 "GB" bullet of
`docs/superpowers/specs/2026-07-13-graph-modality-design.md` on top of GA's
frozen interfaces (`anodyne_graph.models`, `serialization`, `LLMGraphGenerator`,
`ontology`) — no GA signature changed. Adds four generation engines plus a
constraint-validation layer, all producing a `GraphDataset` via the standard
`.generate(spec, start_index, count, seed, shard_index=0)` shape and selected by
`spec.directives`.

## New modules (`packages/anodyne-graph/src/anodyne_graph/`)

- `properties.py` — shared, seeded PII-faking + datatype synthesis (`is_pii`,
  `fake_pii`, `synthesize`, `coerce`, `synthesize_node_properties`,
  `ontology_from_spec`). Keeps the PII invariant in one place. GA `generator.py`
  left untouched (its private copies remain).
- `topology.py` — `ProceduralTopologyGenerator` (no LLM). networkx models via
  `directives["topology"]`: `barabasi_albert` (m), `watts_strogatz` (k,p),
  `erdos_renyi` (p), `stochastic_block_model` (blocks/sizes, p_in, p_out), `lfr`
  (mu, average_degree, min_community). Seed → `np.random.default_rng([seed,
  shard_index])` → integer handed to networkx. Nodes projected onto ontology
  node types (communities → types for SBM/LFR, else round-robin); edges oriented
  onto the first compatible edge type (domain/range), unmappable edges dropped
  and counted. **LFR runs in a killable spawn subprocess with a wall-clock
  timeout** — networkx LFR is known to hang (ignoring `max_iters`) for some
  seeds; hung attempts are abandoned and the next deterministic seed tried.
- `hybrid.py` — `HybridGraphGenerator(provider, model_config)`. Builds structure
  with the topology engine, then one batched temp-0 LLM call fills *non-PII*
  node attributes; PII stays faked. `directives["method"] == "hybrid"`.
- `from_sample.py` — `FromSampleGraphGenerator(sample)` (+ `from_node_link_bytes`
  / `from_graphml_bytes`). Learns degree distribution (Chung-Lu
  `expected_degree_graph`), node-type proportions, per-type attribute pools, and
  edge-type mix; synthesizes a new graph. **Privacy:** faked PII + fresh ids +
  stochastic topology + a **no-verbatim-subgraph** assertion
  (`assert_no_verbatim_subgraph`) over id-independent edge fingerprints. **No
  differential privacy claimed.**
- `constraints.py` — `OntologyConstraintValidator.check` (pure-Python
  domain/range + required/choices/min-max + optional cardinality) and
  `.validate_shacl` (projects graph+ontology to RDF via `rdflib`, validates with
  `pyshacl`). `inject_violations(ds, n, rng)` adds N dangling-edge violations for
  robustness testing; counts land in metrics.
- `engines.py` — `build_graph_engine` / `generate_shard` dispatch by
  `spec.directives` (source→from-sample, method=hybrid, topology, else GA LLM
  default) + `apply_constraints` post-step (inject/validate/SHACL/cardinality).

## Wiring (`anodyne_workflows/handlers.py`)

`GraphHandler.generate_shards` now calls `engines.generate_shard`; only the
LLM-driven engines resolve a provider/model config. From-sample loads its sample
from `directives["sample_uri"]` (node-link JSON or GraphML) — no new DB table.
GA default path unchanged when no directive is set. One-shard contract kept.

## Deps (`anodyne-graph/pyproject.toml`)

`networkx` (BSD), `rdflib` (BSD), `pyshacl` (Apache). Faker/numpy reused.
